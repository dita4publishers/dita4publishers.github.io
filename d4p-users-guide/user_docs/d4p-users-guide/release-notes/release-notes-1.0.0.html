Release Notes: Version 1.0.0RC00 (github version)

Version 1.0.0RC00

Theme for the Release

This release has the DITA for Publishers project moved from SourceForge to GitHub. The DITA for Publishers project is now the GitHub organization "dita4publishers": http://dita4publishers.github.com. The original SourceForge project will be maintained until most or all D4P users have migrated to the GitHub version.

For the move the GitHub the code has been promoted to the 1.0.0 version, reflecting the maturity of the code and a significant renaming of the plugins and related materials.

As part of this move the following changes have been made:
  • All plugins have been renamed to "org.dita4publishers.*" from "net.sourceforge.dita4publishers".
  • The URNs for all D4P vocabulary modules have been changed to use "dita4publishers.org" rather than "dita4publishers.sourceforge.net". A set of catalogs that map the old SourceForge URIs to the new github versions is provided to minimize disruption to documents and shells that refer to the SourceForge URIs. Shells and documents should be updated as soon as possible.
  • Each plugin and the sample data has been made into its own github git repository. The main dita4publishers project references each of the plugins as a git submodule.
  • The HTML5 plugin code has been updated and refined. All future HTML features will be found hereâ€”the Sourceforge version of the HTML5 plugin is obsolete and should not be used.
  • All of the HTML-based transforms have been updated to take any plugin-provided Ant property extensions for the base XHTML transform. This makes general XHTML extension plugins that require custom Ant parameters work with all the D4P HTML-based transforms (EPUB, Kindle, HTML2, HTML5).
  • The dita-support-lib.xsl and relpath_utils.xsl libraries have been moved to a new DITA Community plugin managed in the DITA Community github organization (http://dita-community.github.com). Also used as a git submodule from the main dita4publishers project.
